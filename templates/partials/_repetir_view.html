A projeção é gerada aplicando um rule chain em ordem fixa sobre cada security do processedPosition da data de origem. As regras compartilham um contexto comum (transactions, security metadata, preços) — cada regra só vê o estado já transformado pelas anteriores. As regras estão implementadas em pages/repetir_posicoes.py (objeto _RULES).

1. RULE_TARGET_PROCESSED_PRICE

Quando processedPosition já existe para a data alvo, o PU dessa snapshot vence sobre qualquer cálculo. É a fonte autoritativa. Sem entrada na target → no-op (próxima regra entra). Adiciona a flag targetPrice.

2. RULE_CURVA_PRICE

Fallback de PU: consulta o template salvo em "Preços na Curva" (data/precificacao_lists.json) usando a chave estrita walletId|securityId|targetDate. Sem fallback agnóstico — entradas sem walletId são ignoradas. Pulada quando targetPrice já marcou a linha. Adiciona a flag curvaPrice.

3. RULE_BUYSELL_QTY (transações)

Soma os delta das transações buySell liquidando na data alvo, com sinal invertido em relação ao cash flow (compra em caixa = negativo, mas a posição aumenta).

  • Se tx.quantity existe → delta = -tx.quantity.
  • Senão, se tx.balance existe e row.pu > 0 → delta = -tx.balance / row.pu.
  • Senão, transação é ignorada (sem info suficiente).

Resultado: new_qty = row.qty + Σ delta, balance = new_qty × pu. Adiciona a flag buySell e anexa cada transação ao bloco "Transação" da prévia.

4. RULE_MATURITY (vencimento)

Quando security.maturityDate == targetDate, a quantidade vai a zero e o saldo também. A linha é mantida no payload (com qty 0) para o upstream registrar a baixa explicitamente — não é removida. Vence sobre buySell na mesma data. Adiciona a flag maturity.

5. RULE_PRICE (tagging final)

Apenas etiqueta o tipo de precificação herdado quando nenhuma das regras de preço anteriores marcou a linha: pricingType B1 → priceUnchanged; C3 → pendingPrice; outros → priceUnchanged.

Caixa (`cashAccounts`)

caixa(target) = caixa(source) + Σ transactions.balance onde liquidationDate ∈ (sourceDate, targetDate] (qualquer tipo, trashed != true). Inclui rendimentos, taxas, depósitos/retiradas — mesma lógica do Painel de Controle > Carteira. Aparece na linha "Caixa" da prévia (banda verde) e vai como Caixa = "Sim" no .xlsx de upload.

Provisões

Lista as provisions cuja janela [initialDate, liquidationDate) cobre a data alvo. Cada provisão aparece como uma linha separada na prévia (banda âmbar). Display-only: não vai no upload.
